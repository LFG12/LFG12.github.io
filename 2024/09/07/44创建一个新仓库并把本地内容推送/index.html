在github创建一个新仓库并把本地内容推送

发表于2024-09-07|更新于2024-09-08
|字数总计:429|阅读时长:1分钟|阅读量:

1 写在前面：本篇文章来给git新手(包括我)来写一个操作流程记录，需要有轻微的git基础，0基础者请查阅更详细的介绍

推荐：git快速上手教程 bilibili迷斯特航

环境：windows10

  1. github上创建一个新的空仓库

​ 可以不自动添加readme.md

​ 然后在本地路径下克隆自己新建的仓库

克隆后的文件夹一般是会有.git文件，不用动它

  1. 克隆后可以在本地路径下看到一个新文件夹

  2. 进入这个文件夹后 打开git bash

  3. 添加自己的文件后，开始进行推送

  4. 将全部文件都加到暂存区

  5. 提交暂存区文件到本地仓库【 git commit -m “自己写的提交信息”】

  6. 关联远程仓库：（xxx为远程仓库地址）

  7. 推送到远程库【git push -u origin xxx master】（后面这个master是分支名,xxx是仓库路径）

或者也可以参照github官方给出的示例：

xxx为你远程仓库的地址

在以及创建完分支，成功推送过一次以后，再进行添加文件操作就更简单了
